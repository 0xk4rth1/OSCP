
**Subdomain Fuzzing using ffuf:**

`ffuf -w /usr/share/wordlists/seclists/Discovery/DNS/subdomains-top1million-20000.txt -H "Host: FUZZ.connected.htb" -u http://connected.htb/ `

(it will enumerate subdomains with the wordlist, we can apply match and sort filters to get effective results)

`Subdomain Fuzzing using ffuf:

`ffuf -w /usr/share/wordlists/seclists/Discovery/DNS/subdomains-top1million-20000.txt -H "Host: FUZZ.connected.htb" -u http://connected.htb/ `-mc 200`

It will match the status code of 200.

**Directory Bruteforcing using gobuster:**

For finding common paths:

`gobuster dir -u "http://connected.htb/" -w /usr/share/wordlists/seclists/Discovery/Web-Content/common.txt`  

If an application uses php, then we can add extension to get more valid files

`gobuster dir -u "http://connected.htb/" -w /usr/share/wordlists/seclists/Discovery/Web-Content/common.txt -x php`
