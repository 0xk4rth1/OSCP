The Microsoft Remote Procedure Call (MSRPC) protocol, a client-server model enabling a program to request a service from a program located on another computer without understanding the network’s specifics, was initially derived from open-source software and later developed and copyrighted by Microsoft.

The RPC endpoint mapper can be accessed via TCP and UDP port 135, SMB on TCP 139 and 445 (with a null or authenticated session), and as a web service on TCP port 593.

Enumerating port 135 (Microsoft RPC Endpoint Mapper / MSRPC) helps identify exposed services (like Exchange, DCOM, and Print Spoolers)

To Query Active Endpoints, services and running applications via the Endpoint mapper

`impacket-rpcdump <ip>`   (this maps all active RPC Services exposed to the network)

To pull a list of users from  the remote target (SAM Enumeration)

`samrdump.py <ip>`

Interactive RPC enumeration:

`rpcclient -U "" <ip>`       (connect anonymously)

`rpcclient -U "DOMAIN/username:password" <ip>`    (connect with credentials)
 
Once connected, enumerate users using
 
`rpcclient $> enumdomusers`             (To enumerate domain users)

`rpcclient $> querydispinfo`           (To get a more formatted list of users)
